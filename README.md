Документация для WinForms приложения авторизации на C#
1. Введение
**Название приложения:** WindowsFormsApp7
**Платформа:** Windows (WinForms, .NET Framework / .NET 6+)  
**Цель создания:** Предоставить безопасную систему входа для пользователей с ролями «администратор» и «пользователь», а также обеспечить управление пользователями для администратора.

## 2. Архитектура и основные компоненты
- **База данных:** SQL Server (управляется SQL Server Management Studio 21)  
- **Язык программирования:** C#  
- **Фреймворк:** WinForms  
- **Основные функции:** авторизация, роль разделения, управление пользователями, блокировка аккаунтов.

## 3. Функциональные требования
### 3.1. Экран авторизации
- Поля для ввода:
  - Логин (обязательно)
  - Пароль (обязательно)
- Кнопка «Войти»
- Обработка ошибок:
  - При неверных данных — сообщение:  
    `"Вы ввели неверный логин или пароль. Пожалуйста, проверьте ещё раз введённые данные"`  
  - После 3 неудачных попыток (или неправильного ввода пароля) — блокировка аккаунта.  
  - В случае заблокированного аккаунта — сообщение:  
    `"Вы заблокированы. Обратитесь к администратору"`

### 3.2. Регистрация и управление пользователями
- Для роли «администратор» реализовать интерфейс для:
  - Добавления новых пользователей
  - Изменения данных существующих пользователей
  - Снятия блокировки
- Проверка наличия пользователя при добавлении.
- Сообщение, если логин уже есть в базе.

### 3.3. Роли пользователей
- **Администратор:** управление пользователями, просмотр статистики.
- **Пользователь:** вход, просмотр профиля, изменение своих данных.

## 4. Реализация
- Проверка наличия логина и пароля.
- При успешной авторизации:
  - Вывод сообщения: `"Вы успешно авторизовались"`.
  - Открытие соответствующей роли формы.
- При ошибке — увеличение счетчика ошибок, блокировка по достижении лимита.

## 5. Обработка ошибок и безопасность
- Ввод данных проверяется на обязательность.
- В случае блокировки — доступ к входу запрещен, выводится сообщение.

## 6. Инструкция для разработчика
### 6.1. Структура проекта
- `MainForm` — форма авторизации.
- `AdminForm` — форма управления пользователями.
- `UserForm` — форма профиля.
- `Models` — модели данных, например, `User`.

### 6.2. Основной алгоритм авторизации
1. Проверка логина и пароля в базе.
2. Если пользователь заблокирован — сообщение о блокировке.
3. Если все верно — вход и отображение главной формы.
4. В случае ошибок — увеличение счетчика ошибок, блокировка по достижении лимита.
